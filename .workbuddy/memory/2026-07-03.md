### Watchdog 死锁修复 — Connection.method_60924

**崩溃分析**（crash-2026-07-03_11.18.00-server.txt）：

死锁链路：
1. Server thread 持有 `SynchronizedRandomAccessList` 锁（`ServerConnectionListener.connections` 列表）
2. 新玩家加入触发 `PlayerManager.onPlayerConnect` → `PlayerManager.remove`（踢掉同 UUID 旧玩家）→ `Connection.handleDisconnection` → `method_60924` → `ChannelFuture.awaitUninterruptibly()` **同步阻塞等待** Netty IO 完成写操作
3. Netty Epoll Server IO 线程需要获取同一把 `SynchronizedRandomAccessList` 锁（`initChannel` 中添加新 channel），但锁被 Server thread 持有 → **BLOCKED**
4. Server thread 等 Netty IO → Netty IO 等 Server thread 释放锁 → 经典死锁 → Watchdog 60s 超时杀服

触发条件：服务器满员（32/32）时新玩家尝试连接，同时有同名/UUID 冲突玩家被踢。

**修复方案**：新建 `ConnectionDeadlockFixMixin.java`，使用 `@WrapOperation` 拦截 `Connection.method_60924` 中的 `ChannelFuture.awaitUninterruptibly()` 调用，改为直接返回 future 不阻塞。断开包已写入 Netty pipeline，异步处理即可，无需同步等待。

- 新建：`src/main/java/io/wifi/starrailexpress/mixin/network/ConnectionDeadlockFixMixin.java`
- 注册：`starrailexpress.mixins.json` 中新增 `network.ConnectionDeadlockFixMixin`
- 编译：无新增错误（14 个预存 REINCARNATOR 错误与本次修改无关）
- 注：Krypton 0.2.8 已安装但未覆盖 1.21.1 中 `method_60924` 这个特定代码路径
