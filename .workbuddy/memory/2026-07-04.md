## 2026-07-04: 地图配置添加 ban 职业功能

为地图配置添加了 `bannedRoles` 字段，支持在地图中禁用特定职业，并在地图介绍界面显示。

### 修改的文件
- **AreasWorldComponent.java** — 添加 `bannedRoles` (HashSet<String>) 字段 + getter + NBT 序列化/反序列化
- **MapManager.java** — saveCurrentMapWithoutTry/loadMap 中添加 `bannedRoles` 的 JSON 保存/加载
- **MapIntroduceScreen.java** — 添加 `addBannedRoles()` 和 `roleName()` 方法，在地图属性页显示禁用职业
- **SREDisableManager.java** — 添加 `setMapBannedRoles()` / `clearMapBannedRoles()` 静态方法，在 `isRoleDisabled()` 中检查地图级禁用
- **GameUtils.java** — `initializeGame()` 中应用地图禁用职业（角色分配前），`stopGame()` 中清除
- **zh_cn.json / en_us.json** — 添加 `map_intro.property.banned_roles` 翻译键

### 数据流
地图配置 JSON → AreasWorldComponent.bannedRoles → NBT 同步 + 地图介绍显示 → GameUtils 开局时注入 SREDisableManager → RoleAssignmentPool 建池时过滤

### 修复冤魂（WraithAssassin）突击仍无法击杀低SAN玩家

**根因**：`tickFloatingTargets()` 使用 `GameUtils.killPlayer()`（forceDeath=false），漂浮处决被 `AllowPlayerDeathWithKiller` 等死亡事件拦截，尽管 WraithAssassin 自己的 handler 放行了 `wraith_assault` 死因，但其他 handler（如 role.allowDeath）仍可能拦截。

**修复**：`tickFloatingTargets()` 中 `GameUtils.killPlayer()` → `GameUtils.forceKillPlayer()`，漂浮处决直接绕过所有死亡事件检查，与"处决"语义一致。

**连带修复**：
- `useAssault()` SAN 检查提到 `!isManifested()` 外面，无论显现状态都只命中 SAN < 20 的玩家
- 原 `AllowPlayerDeathWithKiller` handler 中的维度检查保留，作为非漂浮处决路径的安全网
