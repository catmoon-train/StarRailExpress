## 新增 COD 风格字幕报幕系统 (`net.exmo.sre.subtitle`)

**需求**: 在 `net.exmo.sre` 包下添加类似于使命召唤 (CoD) 风格的文字报幕系统，通过网络包指令控制。

**新增文件**:
- `net/exmo/sre/subtitle/SubtitleS2CPayload.java` — S2C 网络包，支持主标题、副标题、持续时间、颜色、打字机效果
- `net/exmo/sre/subtitle/SubtitleCommand.java` — `/sre:subtitle` 命令 + 静态 API（sendToPlayer/sendToAll）
- `net/exmo/sre/subtitle/client/SubtitleHUD.java` — 客户端 HUD 渲染器，队列化播放，渐入/保持/渐出动画

**修改文件**:
- `SRE.java`: 注册 PayloadType (S2C) + 注册命令
- `SREClient.java`: 注册客户端接收器 + HUD 渲染回调 + tick 回调 + disconnect 清理

**功能特性**:
- 队列化字幕：多个字幕按顺序逐个播放，不会互相覆盖
- 渐入→保持→渐出 动画曲线
- 可选打字机逐字显示效果
- 半透明黑色背景底衬
- 主标题(2.2x) + 副标题(1.1x) 双层显示
- 支持富文本/翻译键 Component + 自定义颜色
- 命令 /sre:subtitle <player> <mainText> [subText] [duration] [color] [typewriter]

## 手雷爆炸逻辑重写：空间视线模拟 + 深度计算

**需求**: 重写手榴弹的爆炸判定，用视线模拟和空间深度计算替代原版的畸形判断。

**修改文件**:
- `GrenadeEntity.java` — 全面重写爆炸判定引擎
- `C4Detonation.java` — 复用新的视线检测，移除旧的 unobstructed 方法

**改动详情**:

| 改动项 | 旧逻辑 | 新逻辑 |
|-------|--------|--------|
| 视线检测 | `Explosion.getSeenPercent()` 随机射线，二值判断 | 确定性 26+ 采样点多射线空间模拟 |
| 可见性阈值 | `== 0.0` 完全免疫/完全可见 | 15% 阈值渐进判断 |
| 爆炸层数 | 3 层 (y+0, y+0.5, y-0.5) | 5 层 (0, ±0.5, ±1.0) |
| 实体采样 | 单一随机射线 | 覆盖 8顶点+6面心+12边中点+眼位+体心 |
| 深度计算 | 无 | 精确统计遮挡比例 |
| 射线方式 | Mojang 随机采样 | 确定性 `ClipContext` 方块碰撞 |

**新增公共 API**:
- `GrenadeEntity.computeSpatialVisibility(Level, Vec3, Entity)` → 返回 [0,1] 可见性比例
- `GrenadeEntity.hasExplosionLineOfSight(Level, Vec3, Entity)` → 双射线快速判定（眼+身体）

**C4 改动**: `C4Detonation.hasExplosionLineOfSight()` 改为调用 `GrenadeEntity.hasExplosionLineOfSight()`，删除旧的私有 `unobstructed()`。
如需更精确的 C4 判定，可改用 `computeSpatialVisibility` + 阈值判断。

**编译状态**: ✅ 通过，修改的两个文件无新增错误

**需求**: 如果玩家在短期内多次击杀队友（可排除中立职业和手雷），则执行一个 mcfunction。

**新增文件**:
- `game/TeamKillViolationHandler.java` — 违规检测处理器

**修改文件**:
- `SREConfig.java` — 新增 `teamkill` 分类配置项:
  - `teamKillViolationEnabled` (boolean, 默认 true)
  - `teamKillViolationThreshold` (int, 默认 2, 窗口内队友击杀次数阈值)
  - `teamKillViolationWindowSeconds` (int, 默认 60, 检测时间窗口秒数)
  - `teamKillViolationMcFunction` (String, 默认 "starrailexpress:teamkill_violation")
- `SRE.java` — `registerEventHandlers()` 中调用 `TeamKillViolationHandler.registerEvent()`

**实现逻辑**:
1. 监听 `OnTeammateKilledTeammate.EVENT`
2. 排除条件: 击杀者或受害者任意一方为中立 (`isNeutrals()`) → 跳过
3. 排除条件: 死亡原因路径包含 "grenade" → 跳过

5. 每次记录后清理过期记录 (now - windowSeconds)
6. 达到阈值 → 执行 `mcfunction` + 清空该玩家记录
7. 游戏结束时 (`OnGameEnd.EVENT`) 清空全部记录

## 修复 Phantom 技能一按就取消 + 技能系统优化

### Bug 根因
Phantom 的 toggleable 隐身技能一按就取消：当客户端发送 PRESS 包后松开按键，下一个 tick 发送 RELEASE 包。在 `RoleSkill.useUnified()` 中，RELEASE 阶段仍会调用 handler，此时 `skillReady=false`（因为冷却已由 PRESS 设置），toggleable 的 handler 进入"关闭"分支，**立即移除刚添加的隐身效果**。

同样受影响的 toggleable 技能：Hoan Meirin（飘浮）、Blood Feudist（效果切换）。

### 修复内容

**1. RoleSkill.useUnified() — RELEASE 阶段提前返回**
- 在 HOLD 检查之后、skillReady 检查之前，新增 RELEASE 阶段直接 `stopCasting()` + `return false`
- RELEASE 只对持续施法(continuous)技能有意义，用于停止施法；对非持续技能（包括 toggleable）不应调用 handler
- 移除了后续对 RELEASE 的冗余 `else if (phase == Phase.RELEASE)` 分支（已不可达）
- 简化了 PRESS 后的 cooldown/charges/castCount 逻辑（不再需要 `if (phase == Phase.PRESS)` 包裹）

**2. Definition 新增 `showOnHud` 字段**
- `Definition` record 新增第10个字段 `boolean showOnHud`（默认 `true`）
- `Builder` 新增 `showOnHud(boolean)` 方法
- HUD 渲染时通过 `filter(RoleSkill.Definition::showOnHud)` 过滤不显示的技能

**3. UnifiedSkillHud 全面重写**
- 卡片式布局，半透明背景 + 选中指示点
- 不显示 castCount（使用次数）
- 仅当 maxCharges > 0 时显示"还可使用X次"
- 被动技能显示风格统一
- 使用 `FakeGuiGraphics` 替代 `GuiGraphics` 以匹配项目渲染管线

**4. PhantomHud 精简**
- 移除与 UnifiedSkillHud 重复的冷却/就绪状态显示
- 仅保留隐身激活时的剩余时间 + 按键提示

**5. 翻译键新增**
- `hud.sre.skill.charges_remaining`: zh_cn="还可使用%s次", zh_tw="還可使用%s次", en_us="%s left"
- `hud.sre.passive_label`: zh_cn="被动 ", zh_tw="被動 ", en_us="Passive "
