# 更好的轮抽方案
闪电轮选模式 Lightning Draft Mode
---

## 第一部分：核心规则（无举例纯描述）

### 1. 前置条件
- 职业池已预先抽取完毕，职业总数 **等于** 玩家人数 `M`，且该池已满足所有绑定/对立规则（无需再校验）。
- 玩家已按**阵营权重**降序排列（同权重随机），得到全局顺序队列 `Q`。

### 2. 轮次生成（严格依循示例序列）
每轮开始时，设当前剩余职业数为 `n`（初始为 `M`），剩余玩家数为 `m`。

- 计算本轮参与玩家数 `b = max(1, floor(n / 3))`。
  - 当 `n ≥ 3` 时，`b = ⌊n/3⌋`，且由于 `n ≥ 3b`，有足够的职业为每位玩家提供3个候选选项。
  - 当 `n < 3` 时，`b = 1`，此时候选选项数视 `n` 而定：`n=2` 时给2个选项，`n=1` 时给1个选项。
- 从剩余职业池 `L` 中随机抽取 `need = min(n, b × 3)` 个职业（通常 `need = b×3`，但当 `n < b×3` 时取全部）。
- 将这 `need` 个职业随机打乱，并**平均分配**给 `b` 位玩家，每人分得**最多3个**职业（最后一份可能少于3个，但只在 `n<3` 时发生）。
- 未参与本轮抽取的 `n - need` 个职业**保留**在池中，供后续轮次使用。

### 3. 并行选择与锁定
- 本轮 `b` 位玩家同时看到自己的候选列表（一般为3个，偶尔为2或1个）。
- 每位玩家可以：
  - **手动点击**列表中的任意一个职业，该职业立即锁定归该玩家，并从全局池 `L` 中永久移除。
  - **点击“随机”按钮**，表示放弃候选，系统将在**所有手动选择完成后**，从当前的全局剩余池（包括本轮未被抽出的职业和未被选中的候选职业）中**随机抽取一个职业**分配给该玩家，并从池中移除。
- 限时规则：每位玩家的可用时间为 `（其选项数）× 3秒`，所有玩家同时开始计时，当所有玩家提交完毕后立即进入下一轮。

### 4. 轮次迭代与终止
- 一轮结束后，更新剩余职业数 `n`（减去已分配人数）和剩余玩家队列。
- 重复上述过程，直到 `n = 0` 且 `m = 0`。
- 整个分配过程严格对应您的序列：若 `M=24`，则依次经历人数 `8,5,3,2,2,1,1,1,1`，职业剩余数 `16,11,8,6,4,3,2,1,0`。

## 第二部分：特例演示（24人，严格匹配序列）

### 设定
- 玩家顺序：`P1 ~ P24`。
- 职业池（已打乱）：`C1 ~ C24`。

### 轮次逐轮详解

| 轮次 | 剩余职业数 n | 本轮人数 b | 抽取职业数 | 每人选项数 | 操作说明 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| 第1轮 | 24 | ⌊24/3⌋=8 | 24 | 每人3个 | 取全部24个职业，分8份，每份3个，P1~P8各得3选项。 |
| 第2轮 | 16 | ⌊16/3⌋=5 | 15 | 每人3个 | 从16个中抽15个，分5份，P9~P13各3个，留1个未抽。 |
| 第3轮 | 11 | ⌊11/3⌋=3 | 9 | 每人3个 | 抽9个，分3份，P14~P16各3个，留2个。 |
| 第4轮 | 8 | ⌊8/3⌋=2 | 6 | 每人3个 | 抽6个，分2份，P17~P18各3个，留2个。 |
| 第5轮 | 6 | ⌊6/3⌋=2 | 6 | 每人3个 | 抽6个，分2份，P19~P20各3个，留0个（全部抽出）。 |
| 第6轮 | 4 | ⌊4/3⌋=1 | 3 | 1人3个 | 抽3个，P21得3个，留1个。 |
| 第7轮 | 3 | ⌊3/3⌋=1 | 3 | 1人3个 | 抽3个，P22得3个，留0个。 |
| 第8轮 | 2 | ⌊2/3⌋=0 → 取1 | 2 | 1人2个 | 抽2个，P23得2个，留0个。 |
| 第9轮 | 1 | ⌊1/3⌋=0 → 取1 | 1 | 1人1个 | 抽1个，P24得1个，留0个。 |

### 具体操作示例（以第2轮为例）
- 本轮剩余16个职业，参与玩家：P9~P13共5人。
- 从16个职业中随机抽取出15个（留1个未抽），将这15个随机分成5组，每组3个，分别给P9~P13。
- 假设P9~P13各自选择（或随机）：
  - P9 手动选 C5
  - P10 选“随机”，系统在手动选择完成后，从剩余池（包含刚才未抽的1个 + 本轮抽取但未被选的14个）中随机抽一个给P10，例如抽到 C12。
  - P11 手动选 C8
  - P12 手动选 C2
  - P13 手动选 C15
- 本轮共分配5个职业，剩余职业数变为16-5=11，符合序列。

### 随机选项的详细处理
- 当玩家点击“随机”后，该玩家不锁定任何候选职业。
- 在所有手动选择完成并锁定后，系统统计当前全局剩余职业（包含本轮未抽出的和未被选中的候选职业），从中随机抽取一个分配给该玩家。
- 若有多名玩家选择“随机”，则按顺序依次抽取（但抽取时池子会动态变化，每次抽取后移除被抽中的职业）。

### 时间消耗
- 每轮并行计时，最长耗时为当轮最大选项数 × 3秒。
- 前7轮每人选项均为3，所以每轮上限9秒；第8轮6秒，第9轮3秒。
- 总最坏耗时 = 7×9 + 6 + 3 = 72秒。但实际因玩家快速提交，通常远小于该值。

## 方案特点总结
| 特性 | 实现方式 |
| :--- | :--- |
| **严格匹配示例序列** | 每轮人数 = max(1, floor(n/3))，完全复现您给的数字链。 |
| **尽量3个选项** | 除最后两轮外，所有玩家均有3个候选，仅最后2位玩家分别获得2个和1个选项。 |
| **并行加速** | 每轮多人同时选择，总轮数固定为9轮，与人数无关。 |
| **选择即最终** | 手动选择或随机抽取后立即锁定，绝不二次分配。 |
| **随机灵活** | 随机从全局剩余池抽取，给予玩家意外惊喜。 |
| **公平顺序** | 靠前玩家优先参与，且始终拥有3个选项。 |

此方案完美满足您“尽量抽取三个”的核心诉求，同时保留了您原始数字序列的简洁数学规律，实现简单、体验流畅。
