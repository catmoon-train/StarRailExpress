# 志愿征召模式
英文：Voluntary Draft Mode（简称 VD 模式）

---

- **第一部分（无举例纯规则）**：全流程的数学定义与算法逻辑。
- **第二部分（特例演示）**：用具体阵营池和人数模拟完整分配过程。

---

# 第一部分：核心分配规则（无举例纯描述）

## 1. 基础环境定义
- **职业池**：固定包含编号 `1` 至 `24` 的二十四种职业。
- **名额上限（稀缺性）**：
  - 职业 `1` 的单局名额上限为 **2** 个（因绑定规则 `1->1` 要求双份）。
  - 职业 `3` 与职业 `4` 的单局名额上限各为 **1** 个。
  - 除上述特殊职业外，其余所有职业（`2, 5, 6, ..., 24`）的单局名额上限均为 **1** 个。
  - *若当前房间总玩家人数超过所有职业名额上限总和（默认25个），系统自动将职业 `24`（作为绝对中立的安全垫）的名额上限扩容至满足总人数需求。*
- **阵营池**：每个阵营（如狼、中立偏狼、中立、平民、警卫）均拥有一个独立的职业子池（子池为 `1-24` 的子集）。系统严格从玩家所属阵营的子池中为其抽取候选职业。
- **抽取常量（N）**：固定为 **3**。系统为每位玩家从其阵营子池中随机抽取 **3** 个互不重复的职业，构成该玩家的专属志愿表。

## 2. 玩家交互（同时提交）
- 每位玩家需对系统分配给的 **3** 个职业进行 **严格排序**，生成第一志愿、第二志愿、第三志愿。
- 所有玩家在倒计时结束后 **同时提交** 志愿排序。提交后不可更改。

## 3. 阵营权重（无个人权重）
- 每个阵营拥有一个由服务器写死的固定权重数值（整数）。玩家**无法**通过任何手段修改该数值。
- 不同阵营权重可以相等。权重仅代表该阵营在抢夺稀缺职业时的 **宏观优先级**，不与任何游戏内货币或消耗品挂钩。

## 4. 职业分配优先级三重判定法
在分配任何具体职业时，系统对申请该职业的所有玩家进行以下**严格降序排列**（排名越靠前，分配优先级越高）：

1. **第一优先级（志愿顺序）**：第一志愿玩家 > 第二志愿玩家 > 第三志愿玩家。
2. **第二优先级（阵营权重）**：阵营权重数值 **高者** 优先。
3. **第三优先级（随机种子）**：若阵营权重完全相等，系统调用基于玩家ID与当前对局随机种子生成的哈希值进行排序，**完全随机**决定先后。

## 5. 串行分配与即时对立拦截（执行流程）
系统严格按三阶段执行，每个职业的容量一旦占满，后续申请者立即落选并进入下一阶段。

- **第一阶段（第一志愿分配）**：系统遍历所有职业。对每个职业，提取将该职业作为第一志愿的所有玩家，按 **“三重判定法”** 排序，截取该职业名额上限数量的玩家 **锁定**。未锁定的玩家进入待定池。
- **第二阶段（第二志愿分配）**：针对待定池中的玩家，提取其第二志愿职业，同样按 **“三重判定法”** 排序并截取剩余容量锁定。仍未锁定的玩家进入下一轮。
- **第三阶段（第三志愿分配）**：针对剩余玩家，提取其第三志愿职业，按同样规则截取剩余容量锁定。
- **即时对立拦截（硬规则）**：在上述任一阶段的分配过程中，当一个职业即将被锁定给某位玩家前，系统实时检索当前已锁定的职业池。若该职业与池中**任何一个**已锁定职业构成单向对立（例如已存在 `1` 则禁止锁定 `5`），则**立即禁止本次锁定**，该玩家不消耗该志愿名额，直接落选进入下一阶段。

## 6. 绑定规则的强制征召（分配后处理）
三轮志愿分配结束后，系统检查当前最终锁定的职业池。针对单向绑定规则，执行以下**强制征召**（按顺序触发），直至全部满足：

- **规则 `1->1`（双份校验）**：若池中职业 `1` 的数量为 `1`（小于2），系统从所有当前持有**非职业1**的玩家中，找出阵营权重**最低**（若同权则随机）的那一位，**强制剥夺其当前职业**，征召其转为职业 `1`。若池中职业 `1` 数量为0，则按上述方式连续征召两名玩家补齐至2个。
- **规则 `3<->4`（成对校验）**：若池中存在 `3` 但不存在 `4`，系统从所有持非必需职业（非4）的玩家中征召权重最低者强制转为 `4`；反之若存在 `4` 但不存在 `3`，则强制征召权重最低者转为 `3`。
- **规则 `2->9`（互补校验）**：若池中存在 `2` 但不存在 `9`，系统从所有持非职业9的玩家中征召权重最低者强制转为 `9`。
- **补偿触发**：每次强制征召发生后，权重管理器立即向被征召玩家发放高额固定补偿（基础权重值×2），该补偿不影响后续分配。

## 7. 终极兜底（永不落空）
- 若经过上述三轮分配及强制征召后，仍有玩家**未被分配任何职业**，系统直接为该玩家授予 **职业 `0`（百搭自由人）**。
- 职业 `0` 不参与任何绑定与对立规则，名额无限。授予后，权重管理器向该玩家发放超级保底补偿（基础权重值×5）。

# 第二部分：特例演示（带具体数值模拟）

## 设定前提（本局模拟）
- **房间人数**：6名玩家。
- **职业默认容量**：1号容量为2，其余（含3、4）容量均为1，总计25容量，远大于6人，无需扩容。
- **阵营权重表**（由Manager给出）：
  - 狼阵营：`90`
  - 中立偏狼：`70`
  - 警卫阵营：`85`
  - 中立：`60`
  - 平民：`50`

- **阵营职业池子映射**（本局生效）：
  - 狼池：`{1, 2, 5, 11, 12}`
  - 警卫池：`{3, 4, 8, 9, 13}`
  - 平民池：`{6, 7, 10, 14, 15}`

- **玩家列表与抽取结果**（系统随机从各自阵营池抽3个）：

| 玩家 | 阵营 | 阵营权重 | 系统给出的3个职业（排序前） |
| :--- | :--- | :--- | :--- |
| P1 | 狼 | 90 | `[1, 5, 11]` |
| P2 | 狼 | 90 | `[1, 2, 12]` |
| P3 | 警卫 | 85 | `[3, 8, 9]` |
| P4 | 平民 | 50 | `[6, 7, 10]` |
| P5 | 中立偏狼 | 70 | `[2, 5, 11]` |
| P6 | 中立 | 60 | `[7, 10, 14]` |

## 第一步：玩家同时提交志愿排序（人为设定）

| 玩家 | 第一志愿 | 第二志愿 | 第三志愿 |
| :--- | :--- | :--- | :--- |
| P1 | **1** | 5 | 11 |
| P2 | **2** | 1 | 12 |
| P3 | **3** | 8 | 9 |
| P4 | **6** | 7 | 10 |
| P5 | **5** | 2 | 11 |
| P6 | **10** | 7 | 14 |

---

## 第二步：第一阶段（第一志愿分配 + 即时拦截）

- **职业 `1`**：申请者仅P1。容量2，P1锁定 `1`。 ✅（池中：`{1}`）
- **职业 `2`**：申请者仅P2。容量1，P2锁定 `2`。 ✅（池中：`{1, 2}`）
- **职业 `3`**：申请者仅P3。容量1，P3锁定 `3`。 ✅（池中：`{1, 2, 3}`）
- **职业 `6`**：申请者仅P4。容量1，P4锁定 `6`。 ✅（池中：`{1, 2, 3, 6}`）
- **职业 `5`**：申请者P5（第一志愿）。**即时拦截触发**：当前池中已有 `1`，而规则为 `1->5` 对立（即 `1` 出现时不能有 `5`）。系统**禁止**P5锁定 `5`。P5落选，进入待定池。
- **职业 `10`**：申请者P6（第一志愿）。容量1，P6锁定 `10`。 ✅（池中：`{1, 2, 3, 6, 10}`）

**第一阶段结束**：P1(1), P2(2), P3(3), P4(6), P6(10) 已锁定。P5待定。

---

## 第三步：第二阶段（第二志愿分配 + 即时拦截）

- 当前仅剩P5待分配。P5的第二志愿是 `2`。
- 系统检查职业 `2` 的剩余容量：职业2上限为1，且**已被P2锁定**（容量已满）。P5无法分配，继续落选至第三阶段。

---

## 第四步：第三阶段（第三志愿分配）

- P5的第三志愿是 `11`。职业 `11` 容量为1，当前无人锁定，且不与池中职业 `{1,2,3,6,10}` 产生任何单向对立（11无对立规则）。
- **分配成功**：P5锁定职业 `11`。✅

**三轮分配结束**：最终池子为 `{1, 2, 3, 6, 10, 11}`。

---

## 第五步：强制征召（处理绑定规则）

系统检查当前池子 `{1, 2, 3, 6, 10, 11}` 与全局规则：

1. **检查 `1->1`**：池中只有 **1个** 职业 `1`（P1持有），但规则要求必须 **2个**。触发征召。
   - 从持有非1职业的玩家（P2持2, P3持3, P4持6, P5持11, P6持10）中，找出阵营权重最低者。
   - P4（平民权重50）权重最低。强制剥夺P4的职业 `6`，将P4转为职业 `1`。
   - 此时池子变为 `{1(P1), 1(P4), 2(P2), 3(P3), 10(P6), 11(P5)}`。满足 `1` 计数=2。权重管理器向P4发放高额补偿。

2. **检查 `3<->4`**：池中存在 `3`（P3持），但不存在 `4`。触发征召。
   - 从持有非4职业（且非刚需保留者）中找权重最低者。当前最低为 **P6（中立权重60）**，强制剥夺P6的职业 `10`，将P6转为职业 `4`。
   - 此时池子变为 `{1, 1, 2, 3, 4, 11}`。满足3和4成对存在。权重管理器向P6发放补偿。

3. **检查 `2->9`**：池中存在 `2`（P2持），但不存在 `9`。触发征召。
   - 从持有非9职业且未被征召过的玩家中找权重最低者。当前剩余为 **P5（中立偏狼权重70）** 和 **P3（警卫权重85）**。P5权重70更低。
   - 强制剥夺P5的职业 `11`，将P5转为职业 `9`。
   - 此时池子变为 `{1, 1, 2, 3, 4, 9}`。满足 `2->9` 绑定。权重管理器向P5发放补偿。

---

## 最终锁定结果

| 玩家 | 最终职业 | 与原第一志愿对比 | 补偿情况 |
| :--- | :--- | :--- | :--- |
| P1 | **1** | ✅ 保留第一志愿 | 无 |
| P2 | **2** | ✅ 保留第一志愿 | 无 |
| P3 | **3** | ✅ 保留第一志愿 | 无 |
| P4 | **1** | ❌ 第一志愿6被强制征召改职 | 获得高额补偿 |
| P5 | **9** | ❌ 第一志愿5被对立拦截，最终被征召改职 | 获得高额补偿 |
| P6 | **4** | ❌ 第一志愿10被征召改职 | 获得高额补偿 |

**最终职业池 `{1, 1, 2, 3, 4, 9}` 完全满足您给出的所有规则**：
- `1->1`：两个1存在 ✅
- `1` 对立 `5,6`：无5、6存在 ✅
- `5` 对立 `1,7`：无5存在 ✅
- `3<->4`：两者共存 ✅
- `2->9`：2和9共存 ✅
- `8->9` 对立：无8存在 ✅
- `9->10` 对立：无10存在 ✅
