# MURDER 模式经济系统总结 & 创造性改革方案

> 基于源码完整分析 | 目标：局内 MURDER 模式深度重构

---

## 一、当前经济系统全景总结

### 1.1 货币体系

当前 MURDER 模式存在 **4 种价值载体**，但实际局内核心仅 1 种：

| 货币 | 生命周期 | 局内作用 | 存储位置 |
|------|----------|----------|----------|
| **回合金币** | 单局有效 | 唯一局内流通货币 | `SREPlayerShopComponent.balance` (CCA)，每局 `init()` 清零 |
| **皮肤金币** | 永久 | 局外抽卡/解锁修复角色 | MySQL + NBT 持久化 |
| **第四房间金币** | 独立模式 | 第四房间商店专用 | `FourthRoomPlayerState.coins` |
| **动态商店货币** | 基础设施 | 预留多货币支持 | `DynamicShopComponent.coins` Map |

**核心事实**：MURDER 模式的局内经济 100% 围绕回合金币运转，是一种**单一货币、单一消费路径**的线性经济。

### 1.2 回合金币收入来源

| 来源 | 金额 | 频率 | 适用对象 | 代码位置 |
|------|------|------|----------|----------|
| **开局赠送** | 100 (默认) | 一次性 | 杀手阵营 | `SREMurderGameMode.assignRole()` L114-115 |
| **被动收入** | 5 | 每10秒 | 仅杀手阵营 | `GameConstants.PASSIVE_MONEY_TICKER` + `canAutoAddMoney()` |
| **击杀奖励** | 100 (默认) | 每次击杀 | 杀手阵营 | `GameMode.killPlayer()` L714 |
| **手榴弹击杀** | 75/人, 上限375 | 手榴弹击杀时 | 杀手阵营 | `SREConfig.grenadeMoneyPerKill` |
| **大亨修饰符** | 25 | 每60秒 | 特定修饰符持有者 | `MagnatePassiveIncomeHandler` |
| **连击奖励** | 5/级, 上限10 | 连续击杀 | 杀手阵营 | `GameConstants.STREAK_BONUS_PER_LEVEL` |

**10分钟一局的经济总量估算**（杀手视角）：
- 被动收入：60次 × 5 = 300 金币
- 开局赠送：100 金币
- 击杀 2 人：200 金币
- **总计约 600 金币**，不考虑特殊修饰符

### 1.4 商店购买流程详解

```
玩家点击购买 → StoreBuyPayload(C2S) 
  → SREPlayerShopComponent.tryBuy(index)
    → [1] 索引范围验证
    → [2] 手榴弹额外冷却检查 (grenadePurchaseCooldown, 默认30s)
    → [3] 余额 >= 价格?
    → [4] 物品不在冷却中?
    → [5] canDisplay(player) && canBuy(player)?
    → [6] 不在安全时间 (isSafeTime)?
    → [7] entry.onBuy(player) — 默认插入背包，特殊物品触发效果
    → [8] 扣除余额，记录回放
```

### 1.5 购买限制机制

| 限制类型 | 说明 |
|----------|------|
| **安全时间** | 开局初始一段时间内不可购买 |
| **物品冷却** | 每件商品独立冷却时间（5秒~5分钟不等） |
| **手榴弹购买冷却** | 购买后额外30秒冷却（独立于使用冷却） |
| **全局停电冷却** | 停电触发后所有玩家共享冷却 |
| **角色限制** | `canDisplay`/`canBuy` 钩子可按角色过滤 |

## 二、当前系统深度诊断

### 🔴 P0 级问题（破坏游戏体验的根本缺陷）

#### 问题1：经济系统完全是杀手的"私域"，平民被经济排斥

**现状**：`canAutoAddMoney()` 仅对杀手阵营返回 true。平民没有金币收入、没有商店、没有任何经济参与方式。

**影响**：这不仅仅是"不公平"——它直接破坏了社交推理游戏的核心乐趣。在 Murder Mystery 类游戏中，**资源竞争是驱动社交互动的引擎**。当一方完全没有经济参与时：
- 平民无法通过经济行为传递信息（"我买了什么"没有意义，因为平民什么都没买）
- 杀手的经济行为缺乏观测维度（平民无法通过"谁在消费"来判断嫌疑人）
- 游戏沦为纯粹的"跑图+躲藏"循环，对平民来说缺乏策略深度

**当前唯一的平民经济参与**：第四房间模式的独立商店——但这与 MURDER 模式完全割裂。

#### 问题2：线性经济 = 零策略深度

**数学证明**：
- 被动收入恒定（5/10s），不存在"投入更多时间/风险换取更多收入"的选择
- 击杀奖励恒定（100/杀），不存在"风险收益权衡"
- 商店定价静态，不存在"时机选择"
- 没有投资/储蓄/借贷机制，金币唯一的用途就是消费

**结果**：存在一个确定的"最优购买序列"。对于经验丰富的玩家，经济系统已退化为"等时间→按键购买"的机械操作。

#### 问题3：武器性价比严重失衡

| 武器 | 价格 | 击杀效率 | 实际性价比 | 评级 |
|------|------|----------|-----------|------|
| 小刀 Knife | 130 | 近战一击必杀 | 130金币/击杀 | S 级 |
| 左轮 Revolver | 285 | 远程一击 | 285金币/击杀 | B 级 |
| 霰弹枪 Short SG | 300 | 近距离爆发 | 300金币/击杀 | C 级 |
| 手榴弹 Grenade | 330 | AOE + 300s冷却 | 330~82.5/杀(1~4人) | B~C 级 |
| 精神强化 Psycho | 400 | 短暂爆发+1护盾 | 400金币/高失败率 | D 级 |

**老玩家共识**：小刀(130) + 停电(140) = 270 金币是最优组合，远胜单买任何高端武器。

**根本原因**：近战武器在"列车"这种封闭环境中天然优势，而定价没有补偿这种环境优势。

---

### 🟡 P1 级问题（限制游戏上限的设计短板）

#### 问题4：售货机系统是"死系统"

售货机方块（`VendingMachinesBlock`）目前是纯管理员工具：
- 仅管理员可放置和配置
- 商品列表完全静态，NBT 持久化但无动态机制
- 玩家无交互创造（不能经营、不能补货、不能竞价）
- 没有与游戏进程的任何动态联动

**本质**：售货机是"地图装饰"，不是"经济节点"。

#### 问题5：信息具有经济价值但无法交易

在社交推理游戏中，**信息是最稀缺、最有价值的资源**。"谁是杀手""杀手买了什么武器""谁刚从售货机买了东西"——这些信息在理论上具有巨大经济价值，但系统完全没有提供信息交易的机制。

**当前仅有的信息渠道**：
- 死亡公告（免费，延迟）
- 便签（10 金币，但仅限文字，无法携带可验证信息）
- 地图任务（奖励情绪值，非信息）

#### 问题6：经济与胜负条件完全脱钩

金币的唯一用途是购买道具 → 道具的唯一用途是击杀/自保 → 击杀影响胜负。

但经济本身**永远不可能成为胜利条件**。即使一个杀手攒了 1000 金币，如果不能击杀平民，这些金币毫无意义。这让经济系统沦为"辅助系统"而非"核心系统"。

---

### 🟢 被忽视的系统潜力

| 潜力 | 当前状态 | 可能方向 |
|------|----------|----------|
| 阵营经济差异 | 只有杀手有经济 | 平民经济 → 信息/防御/协作 |
| 动态定价 | 完全静态 | 浮动定价 → 时机选择 |
| 玩家间经济互动 | 不存在 | 交易/借贷/悬赏 → 社交驱动 |
| 经济事件 | 不存在 | 限时促销/通胀/安检 → 变数制造 |
| 售货机动态化 | 管理员静态配置 | 可交互经济节点 |
| 经济胜利条件 | 不存在 | 资本博弈模式 |

## 三、创造性巨大改革方案

> 核心设计理念：**"让每一枚金币都是一个决策"**

---

### 🏗️ 改革一：双阵营经济生态

> **目标**：让经济系统成为所有玩家的核心体验，而非杀手的专属工具。

#### 1.1 平民经济体系——"信息与协作经济"

平民没有杀人的需求，但有**找人和自保**的需求。平民经济围绕"信息获取"和"群体防御"构建：

**平民收入来源**：
| 来源 | 金额 | 触发条件 |
|------|------|----------|
| 完成地图任务 | 15~30 | 每完成一个任务 |
| 发现尸体并报告 | 50 | 首次报告一具尸体 |
| 存活到阶段结束 | 20 | 每存活2分钟 |
| 目击可疑行为 | 25 | 目击某人使用武器/开锁 |

**平民信息商店**：
| 商品 | 价格 | 效果 |
|------|------|------|
| 📹 监控录像带 | 40 | 查看指定区域过去15秒内经过的玩家列表 |
| 🔦 足迹追踪器 | 30 | 显示目标玩家最近30秒的行动轨迹（持续10秒） |
| 🛡️ 临时护盾 | 60 | 抵挡一次近战攻击（持续60秒或触发一次） |
| 🚨 紧急按钮 | 50 | 全局广播自己的位置和"请求帮助"（非安全区也可用） |
| 📋 身份扫描仪 | 80 | 高亮显示附近5格内所有持有武器的玩家（3秒） |
| 🤫 悄步药水 | 25 | 消除脚步声60秒 |
| 📡 窃听器 | 35 | 放置后可远程查看该区域玩家列表（持续2分钟） |

#### 1.2 中立角色经济——"对冲经济"

中立角色的胜利条件通常独立于阵营对抗。给他们一个"对冲商店"：

| 商品 | 价格 | 效果 |
|------|------|------|
| 🎲 命运硬币 | 100 | 押注一方阵营获胜，赢取2倍返还 |
| 🕐 时间胶囊 | 150 | 延长游戏时间30秒（中立角色通用需求） |
| 🎭 身份迷雾 | 80 | 隐藏自己的角色类型标识2分钟 |
| 💰 赏金发布 | 60 | 对指定玩家发布悬赏（其他玩家击杀可获得奖励） |

#### 1.3 实现代码路径

```
修改 canAutoAddMoney() → 新增平民/中立收入逻辑
新增 SREPlayerShopComponent 中的阵营判断
新增 ShopContent.civilianEntries / neutralEntries
修改 StoreBuyPayload 支持商店类型字段
```

### 🏗️ 改革二：动态经济引擎

> **目标**：打破线性经济，让"何时买"和"买什么"成为真正的策略决策。

#### 2.1 浮动定价机制

每件商品的价格不再固定，而是受全局因素动态影响：

```java
// 伪代码：价格修正因子
float getPriceModifier(ShopEntry entry, ServerLevel world) {
    float modifier = 1.0f;
    
    // 因素1：稀缺性涨价 — 每次有人购买同类商品，该类商品+10%
    modifier *= (1.0f + purchaseCount.get(entry.type) * 0.10f);
    
    // 因素2：恐慌性降价 — 每次有人死亡，所有武器-8%
    modifier *= (1.0f - deathCount * 0.08f);
    
    // 因素3：时间压力 — 游戏后期折扣加速
    if (gameTime > 8min) modifier *= 0.7f;
    else if (gameTime > 6min) modifier *= 0.85f;
    
    // 限制：价格波动 ±50%
    return Math.max(0.5f, Math.min(1.5f, modifier));
}
```

**设计意图**：
- 早买贵但先手，晚买便宜但没时间用 → **时机博弈**
- 杀手买武器 → 同类武器涨价 → 其他杀手被逼买别的 → **购买路径分化**
- 人死多了 → 武器打折 → 刺激后期消费 → **节奏控制**

#### 2.2 经济事件系统——"列车广播"

每隔固定时间触发一次全局经济事件，增加变数和话题性：

| 事件 | 触发时间 | 效果 | 持续时间 |
|------|----------|------|----------|
| 🚉 **列车到站·补给** | 3分钟 | 所有商品8折 | 30秒 |
| 🔒 **安检升级** | 5分钟 | 武器类商品不可购买 | 45秒 |
| 🎁 **乘务促销** | 4分钟 | 随机赠送所有玩家一件10金币以下的工具 | 即时 |
| 📈 **通货膨胀** | 7分钟 | 所有价格×1.5，但击杀奖励×2 | 60秒 |
| 🌙 **午夜特卖** | 6分钟 | 随机1件高端武器5折（仅限1件） | 20秒 |
| 🚨 **紧急状态** | 存活<4人时 | 所有商品6折，取消安全时间 | 到局末 |

**代码实现路径**：
- 新增 `EconomyEventManager` 类，挂载到 `tickServerGameLoop`
- 新增 `SREGameWorldComponent.economyState` 字段（当前事件、剩余时间）
- 修改 `ShopEntry` 的 `price()` 方法读取全局价格修正

#### 2.3 动态售货机

将售货机从"管理员静态配置"升级为"动态经济节点"：

```
新机制：
- 售货机商品每2分钟自动刷新（从预设池随机抽取）
- 稀有商品出现概率低但价格优惠（黑市商品偶尔出现在售货机中）
- 售货机有库存限制（每件商品限购1~3次）
- 某些售货机需要"硬币"道具才能使用（硬币从任务中获得）

这使售货机成为"地图资源争夺点"，
玩家需要抢时间去特定售货机购买稀有商品。
```

### 🏗️ 改革三：信息经济与社交博弈

> **目标**：让信息成为可交易的商品，驱动社交互动。

#### 3.1 信息悬赏板系统

```
机制：
- 任何玩家可以在"悬赏板"（新方块）上发布悬赏
  例："提供杀手身份的可靠证据 → 悬赏50金币"
- 其他玩家可以提交"证据"（如：我目击了某人购买武器）
- 发布者确认后 → 赏金自动发放
- 虚假情报 → 提交者被罚款赏金的50%，归发布者所有
- 悬赏板全局可见，任何人都可以看到有哪些悬赏

代码实现：
- 新增 BountyBoardBlock + BountyBoardBlockEntity
- 新增 BountyBoardC2SPacket / BountyBoardS2CPacket
- 新增 SREGameWorldComponent.bounties 列表
```

**社交影响**：
- 杀手可以发布虚假悬赏误导平民
- 平民可以通过悬赏验证自己的推理
- 中立角色可以靠"卖信息"赚钱
- 每一笔悬赏都是一次社交博弈

#### 3.2 情报商人角色

```
新中立角色："情报贩子" (Info Broker)
- 专属商店：购买不同精度的探查信息
  - 10金币：获知目标本局是否购买过武器
  - 25金币：获知目标当前是否持有武器
  - 50金币：获知目标的确切角色阵营（杀手/警卫/中立/平民）
- 专属技能：可以花费金币"审问"已死亡玩家的尸体，获取其最后目击的玩家名称
- 专属经济：可以创建"情报终端"，将信息以自定义价格出售给其他玩家
  - 买家支付设定价格
  - 情报贩子获得80%，系统抽税20%
```

### 🏗️ 改革四：黑市系统——高风险高回报

> **目标**：提供一条更强力但更危险的购买通道，制造"贪婪 vs 安全"的抉择。

#### 4.1 黑市入口

```
触发条件（满足任一即可出现）：
1. 地图随机刷新1~2个"黑市商人NPC"（伪装成普通NPC）
2. 停电期间，黑市商品价格-30%
3. 持有"暗号"道具（通过完成隐藏任务获取）

识别方式：
- 黑市商人外观与普通NPC完全相同
- 只有靠近2格内才会出现"交易"提示
- 购买时全局广播"有人进入了黑市"（不公布身份）
```

#### 4.2 黑市专属商品

| 商品 | 价格 | 效果 | 风险/代价 |
|------|------|------|----------|
| 🔫 **消音手枪** | 350 | 击杀不触发死亡公告 | 购买时广播"有人进入黑市" |
| 🎭 **伪装斗篷** | 200 | 假装成平民外观15秒 | 被击杀时掉落，暴露身份 |
| 💉 **毒针** | 250 | 延迟60秒死亡，无声无息 | 如果目标在死亡前完成任务，杀手暴露位置 |
| 🪆 **替身娃娃** | 400 | 抵消一次击杀 | 购买后位置被全局标记5秒 |
| ⚡ **电磁脉冲** | 300 | 30秒内所有监控/窃听器失效 | 所有人收到"EMP激活"警告 |

#### 4.3 嫌疑值系统

```
机制：
- 每次黑市购买积累"嫌疑值"(1~3点，根据商品危险度)
- 嫌疑值 > 5: 玩家间歇性在地图上发光（仅平民可见）
- 嫌疑值 > 10: 全局公告"某玩家有重大嫌疑"
- 嫌疑值每60秒自然衰减1点
- 被击杀后嫌疑值清零

这创造了有趣的博弈：
- 杀手要不要冒险去黑市买更强力的武器？
- 买多了会不会暴露？
- 平民会不会故意去黑市买无害物品来"污染"嫌疑值系统？
```

### 🏗️ 改革五：玩家间经济互动

> **目标**：将经济从"玩家 vs 系统"升级为"玩家 vs 玩家"。

#### 5.1 玩家间交易

```
机制：
- 在安全区域内可以发起面对面交易请求
- 交易内容：金币、道具、信息（文字承诺不可强制执行）
- 双方确认后成交
- 系统抽取5%交易税

限制：
- 杀手不能与杀手交易（防止组队垄断）
- 交易双方5秒内不能攻击对方（安全交易窗口）
- 单局单人最多发起3次交易
- 不能交易角色专属物品
```

#### 5.2 保险系统

```
商店新增工具类商品：
- 🛡️ 人寿保险 (150金币): 你在本局被击杀时，你的队友获得200金币补偿
- 💼 财产保险 (80金币): 死后不掉落已购买但未使用的道具
- ⚔️ 复仇基金 (200金币): 击杀你的玩家会被自动悬赏300金币

设计意图：
- 保险是"对冲死亡风险"的经济行为
- 杀手买了复仇基金 → 杀手需要权衡"击杀收益 vs 被悬赏风险"
- 平民买了人寿保险 → 即使死亡也对团队有贡献
```

#### 5.3 赏金猎人系统

```
机制：
- 某玩家连续击杀2人以上 → 自动生成赏金标记
- 击杀赏金目标 → 获得额外金币 = 基础100 + 目标累计击杀数×50
- 赏金最高500金币封顶
- 赏金目标可以看到自己被悬赏的金额

社交影响：
- 赏金目标成为"移动金矿" → 激进杀手被针对性猎杀
- 赏金目标可以选择"低调行事"减少击杀 → 经济压力改变行为模式
- 平民可以靠击杀赏金目标获得金币 → 给平民主动战斗的理由
```

### 🏗️ 改革六：经济胜利条件——"资本博弈"模式变体

> **目标**：让经济本身成为胜利条件，创造全新的游戏维度。

#### 6.1 模式规则

```
"资本博弈"作为MURDER模式的可选变体：

胜利条件（任一达成即结束）：
- 杀手阵营：击杀所有平民 OR 杀手阵营总资产达到 2000 金币
- 平民阵营：完成所有地图任务 OR 平民营总资产达到 5000 金币
- 中立角色：个人资产达到 3000 金币（独立胜利）

经济机制新增：
- 列车股票系统（见下文）
- 资产 = 手持金币 + 已购买道具折价 + 股票市值
```

#### 6.2 列车股票系统

```
4支概念股，价格每30秒波动一次：
- "星穹能源" (Energy): 与击杀数正相关（越混乱越涨）
- "平安保险" (Safety): 与存活人数正相关（越和平越涨）
- "开拓建设" (Build): 与任务完成数正相关
- "量子科技" (Quantum): 纯随机波动（高风险高回报）

交互方式：
- 地图内放置"股票终端"方块
- 买入/卖出实时结算
- 收益直接计入回合金币
- 死亡后股票清零（风险！）

策略深度：
- 杀手想买能源股？那得杀人才能让它涨 → 经济动机驱动游戏行为
- 平民想买平安股？那得保人才能让它涨 → 防守也有经济回报
- 量子科技是纯赌博 → 给中立角色一个博弈空间
```

## 四、武器价格重平衡建议

当前最大的平衡问题是**小刀性价比远超其他武器**。建议重新设计定价逻辑：

### 4.1 新定价原则

```
定价 = 基础价值 × 环境修正 × 使用难度修正

环境修正（列车封闭空间）：
- 近战武器：×1.3（封闭空间优势）
- 远程武器：×0.9（走廊限制射界）
- AOE武器：×1.0（标准）

使用难度修正：
- 一击必杀：×1.2（低风险高回报需要更高价格）
- 需要瞄准：×0.85
- 延迟生效：×0.8
- 自伤风险：×0.75
```

### 4.2 建议新定价

| 商品 | 当前价格 | 建议价格 | 变化 | 理由 |
|------|----------|----------|------|------|
| 鞭炮 | 10 | 10 | → | 合理 |
| 便签x4 | 10 | 10 | → | 合理 |
| 撬棍 | 35 | 40 | ↑ | 开门刚需，微涨 |
| 开锁器 | 80 | 65 | ↓ | 隐秘但180s冷却过长，降低价格补偿 |
| 裹尸袋 | 100 | 120 | ↑ | 处理尸体对杀手极重要，涨价 |
| **小刀** | **130** | **180** | **↑** | **核心调整：一击必杀在列车中太强** |
| 停电 | 140 | 160 | ↑ | 环境控制价值被低估 |
| 左轮 | 285 | 240 | ↓ | 射界限制+需要弹药，适当降价 |
| 霰弹枪 | 300 | 260 | ↓ | 近战爆发但射程极短，降价 |
| 手榴弹 | 330 | 350 | ↑ | AOE潜力大，微涨 |
| 精神强化 | 400 | 320 | ↓ | 当前性价比太低，大幅降价 |

**核心改动**：小刀 130→180 大幅涨价，左轮/霰弹枪/精神强化 降价，拉平性价比差距。

## 七、风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 经济膨胀（金币过多） | 破坏平衡 | 价格波动上限 ±50%，交易税5%，通胀事件限时 |
| 信息滥用（组队利用情报） | 破坏公平 | 信息有延迟、有成本、可能虚假；情报贩子有冷却 |
| 黑市过强（杀手买消音手枪无解） | 平民体验差 | 黑市购买广播+嫌疑值系统+库存限制 |
| 交易欺诈 | 玩家体验差 | 安全交易窗口+次数限制+5秒攻击冷却 |
| 复杂度爆炸 | 学习成本高 | Phase 分期实施，每期上线后收集数据再决定下一步 |
| 股票赌博成瘾 | 伦理风险 | 标记为"娱乐模式"变体，非竞技主模式 |
