## 1. 总体思路

- 你只需**各搭一份**模板：**住宅区** / **避难所** / **搜索区**。
- 开局时系统会为**每支队伍（4 人一家）**按网格偏移，用 `BlockCopyUtils.copyLayer` 把**住宅与避难所各克隆一份**到独立空间，
  游戏结束再快照还原——**你不用手动复制、不用清理**。
- **搜索区不克隆**：所有队共用你搭的那片原区域——各队玩家外出搜刮会在同一片野外**相遇**（搜打撤对抗的来源）。
- 门 / 物资箱 / 电脑等方块直接建在模板里，随克隆自动复制。
- 配置（模板区域坐标、每队网格偏移、出生点）用 `/sre:60s_area` 命令登记，落盘到世界存档目录 `sixty_seconds_config.json`。

```
模板（你搭一份）            开局按队克隆（每队一份，独立空间）
┌───────────┐              队0        队1        队2 ...
│ 住宅区模板 │  ──┬─ +网格偏移 ─>  [住宅0]    [住宅1]    [住宅2]
│ 避难所模板 │  ──┘              [避难所0]  [避难所1]  [避难所2]
│ 搜索区模板 │  ──── 不克隆，全队共用原区域（野外相遇）
└───────────┘
```

## 2. 第一步：准备一张 noReset 地图

模板本身就是地图内容，**不能**被列车重置覆盖。开一局前设：

```
/sre:area_manager set noReset true
```

（这样 `GameUtils.startGame` 会直接开始而不做 FullTrainReset。）

## 3. 第二步：手搭三种模板

三份模板**互相隔开足够远**（间距要大于单个模板尺寸，避免克隆时重叠；网格间距见 §4 `grid`）。

### 3.1 住宅区（每队 60s 内搜物资的家）
- 摆放**箱子**（原版 chest / `sre` 物资箱 `sixty_seconds_supply_box`）当物资来源。
- 摆放**一扇避难所门** `sixty_seconds_shelter_door`（准备阶段右键=菜单存物资，**潜行右键=免菜单快速存入**，见 §8）。
- 记下一个**住宅出生点**坐标（玩家开局站这里）。

### 3.2 避难所（每队独立地下室）

> **夜袭刷新位置（重要）**：夜袭感染体现在**首选刷在避难所门的外侧 5~12 格**（按门朝向自动判断外侧）——
> 请在避难所模板的门外留一片可站立的开阔地（院子/走廊），玩家能从屋内看到怪压过来并布防
> （路障/尖刺/铁丝网/哨戒炮）。若门外是实心墙（全封闭地堡），系统自动退回旧行为：刷在探索区的锚点门远处。
- 摆放**床**（睡眠回血用，后续批次）。
- 摆放**电脑**：小游戏任务点方块 `MinigameQuestBlock`（每分钟自动轮换小游戏，见玩法文档）。
- 摆放**避难所门** `sixty_seconds_shelter_door`（外观=列车钢门，2 格高，放置时朝向玩家）。
  门已**通用化**：右键打开统一菜单，按玩家上下文给选项——
  - 准备阶段：**存入携带物资**（潜行右键=免菜单快速存入）；
  - 游戏日在家：**外出探索**（出门隐身 15s、归来冷却 120s）/ **门外事件** / **拜访别队**；
  - 在探索区：**返回住所**。一扇门够用，多摆几扇也无妨（选项相同）。
  - 手持木板/铁锭/修理包右键门=**加固家门**（耐久/等级，防夜袭）。
- 记下**避难所出生点**坐标（准备结束传送到这里；系统会在其**旁边(+X)**自动放一个箱子装入库存物资）。

### 3.3 搜索区（门后的野外搜物资区，**全队共用、不克隆**）
- 搜索区**不会按队克隆**——所有队伍出门都进入你搭的这片原区域，**各队玩家会在野外相遇**（可交火/抢物资）。
- 摆放**物资箱** `sixty_seconds_supply_box`（每日按 loot 表刷新，右键领取；创造右键=编辑 loot 表；每人每天各领一次，互不影响）。
- 摆放**一扇避难所门**（在搜索区里右键它 → 菜单「返回住所」，送回各自队伍的家）。
- 记下**搜索区出生点**坐标（出门探索传送到这里；整块搜索区会被当作限制盒，玩家不能越界）。
- **每队专属出口（推荐）**：用 `/sre:60s_area exit add <x y z>` 在探索区内登记多个出口点——
  每支队伍按序轮转分到一个，出门落在**自己避难所对应的位置**而不是全挤在同一点。

> 旧的门用途（SEARCH/EVENT/VISIT）已废弃——所有门同菜单；`purpose` 属性仅为旧存档兼容保留。

### 3.4 门 ↔ 独立探索区绑定（可选，用绑定工具）

默认所有门的「外出探索」都通往**同一个** `searchzone` 模板（§4 登记的那个）。若你想让**每扇避难所门各自对应一块独立探索区**
（比如一个避难所里多扇门通往不同野外区域），在**搜索区模板内**多搭几块子区域，再用 `sixty_seconds_area_wand` 绑定：

1. `/give @p noellesroles:sixty_seconds_area_wand`，手持它。
2. **手持工具右键一扇避难所门** → 选中它为「待绑定门」。
3. **右键该门专属探索区的一角**（这里就是玩家出门后的**出生点**）。
4. **右键对角** → 完成绑定，自动写入 `sixty_seconds_config.json`。
5. 重复 2–4 绑定更多门。**未选门时潜行右键**可取消当前选择。

> **快速绑定**：选中门后**潜行+右键**探索区里的一个方块 → 一步成型——出生点=点击处上方，
> 区域=以出生点为中心 水平 ±24 格、垂直 -4..+16。适合快速铺量，精确控制仍用两角模式。

- 门坐标记为模板绝对坐标（开局随避难所克隆按队加偏移）；**绑定的探索区不克隆**——所有队共用你圈的那片原区域。
- 未绑定的门回退到全局 `searchzone`；同一扇门重复绑定会**覆盖**旧绑定。
- 玩家被**限制**在其门绑定的那块盒内（越界拉回），归来冷却/隐身与默认一致。

## 4. 第三步：登记配置（命令）

坐标均为**世界绝对坐标**；系统会对每队叠加网格偏移。模板区域填**两个对角**（顺序随意，自动取正序）。

```
# 三种模板区域（两个对角的绝对坐标）
/sre:60s_area template residential <x1> <y1> <z1> <x2> <y2> <z2>
/sre:60s_area template shelter     <x1> <y1> <z1> <x2> <y2> <z2>
/sre:60s_area template searchzone  <x1> <y1> <z1> <x2> <y2> <z2>

# 三个出生点（模板内绝对坐标；建图时自动换算成相对模板 min 的偏移套到每队克隆区）
/sre:60s_area spawn residential <x> <y> <z>
/sre:60s_area spawn shelter     <x> <y> <z>
/sre:60s_area spawn searchzone  <x> <y> <z>

# 每队网格：第一队相对模板的偏移(base) + 每队在 X 轴的额外间距(spacing，须>模板尺寸)
/sre:60s_area grid <baseX> <baseY> <baseZ> <spacing>

# 避难所锚点门（模板内那扇避难所门的绝对坐标）——配合 shelter_at_door 开关，见 §4.1
/sre:60s_area anchor <x> <y> <z>
/sre:60s_area anchor clear

# 查看当前配置是否完整
/sre:60s_area show
```

每次 set 会自动落盘。`show` 显示 `complete=true` 即配置齐全，可开局；
它同时会打出 `shelter_at_door` / `anchor` / `sea_teleport` 三项的当前值。

### 4.1 避难所锚定到探索区的门（`shelter_at_door`，默认**开**）

默认情况下避难所**不再**克隆到远处的网格空间，而是**直接长在本队那扇探索区出口门上**——
玩家推门出去就是探索区，「外出探索」是字面意义的出门，不再是跨空间传送。

- **锚点**：`shelterAnchorDoor`（避难所模板内那扇门） ↔ 本队的**出口门**（§3.4 绑定工具建在**探索区里**的门）。
  建图时避难所整体平移 `出口门 - 锚点门`，两扇门正好重合成一扇。
- **只影响避难所**：住宅区仍按 `grid` 网格克隆（准备阶段结束照常传送进避难所）。
- **一队一门**：出口门按队序号分配。门不够分的队会**回退网格克隆**并在日志告警——建图前请确保
  探索区里绑定的出口门 ≥ 队伍数。
- **没登记锚点** → 整局回退网格克隆 + 告警。`/sre:60s_area anchor` 若发现该坐标不是
  `sixty_seconds_shelter_door` 会警告：锚点处的方块会被克隆盖到出口门上，指错了出口门就没了。
- 关掉它（`/sre:60s shelter_at_door off`）即恢复旧的网格克隆行为。

> **推荐用锚点绑定工具**（`sixty_seconds_anchor_wand`）代替手敲坐标：
> 右键避难所模板内的那扇门 = 登记锚点；右键探索区里的出口门 = **当场预演**这扇门属于第几队、平移量多少、
> 避难所会落到哪个盒、**是否与别队重叠**；潜行右键门 = 清除锚点；右键任意非门方块 = 打印当前状态。
> 重叠在建图时只会写进日志，用工具能在搭图时就发现并挪门。

> 净空照常生效：避难所落进探索区的地形里时，四周 2 格环带 + 上方 12 格会被挖空（局末按快照还原）。

### 4.2 海图航线开关（`sea_teleport`，默认**关**）

控制海图能否**扬帆传送**与**返回住所**。默认关闭时海图退化成**纯导航图**：
岛屿轮廓、解锁迷雾、庇护所与队友点位**照常显示**，但点岛不传送、「返回住所」置灰——
玩家得自己开船出海、自己开回来走门进家。想要一键往返就 `/sre:60s sea_teleport on`。
创造/旁观不受此限制。与 `shelter_at_door` **互不影响**，可任意组合。

## 5. 第四步：启动与验证

```
/sre:60s start [minutes]         # 或 /tmm:start sre:sixty_seconds
```

预期：
0. 开局先**异步分批建图**（屏幕上方显示「生成地图中… x%」进度）——克隆量大也**不会卡死/触发看门狗**；建完才传送并进入准备阶段。
1. 进入 **PREPARATION**（60s）：每队被传送到各自**住宅出生点**；背包按家庭身份限制（**父亲 8 格 / 其余 2 格**，其余屏障覆盖）。
2. 右键住宅/避难所门 → **记录携带物资进库存**（清空携带槽以便继续搜集）。
3. 60s 后进入 **游戏日(第 1/7 天)**：全队传送到**避难所出生点**，出生点旁自动生成**装好库存物资的箱子**；HUD 显示天数与四值+健康。
4. 每天 9.5 分钟（清晨 1 + 白天 6 + 晚上 2.5，末尾 45 秒睡觉时间），共 7 天；第 7 天后结算。
5. 门菜单「外出探索」传送探索区、物资箱搜刮领取、电脑每分钟轮换小游戏、随机事件、拜访/闯入等按玩法文档运作。

## 6. 命令速查

| --- | --- |
| `/sre:60s start [minutes]` | 启动本模式（也可 `/tmm:start sre:sixty_seconds`） |

| `/sre:60s_area exit add <x y z>` / `exit clear` | **OP**：登记探索区内的**每队出口点**（各队按序轮转分配；不配则共用 searchzone 出生点） |
| `/sre:60s backup save/restore/list [玩家]` | **OP**：手动保存/恢复玩家掉线备份（掉线-重连自动触发） |
| `/sre:60s day <1-7>` / `time <morning\|daytime\|night\|sleep>` | **OP**：跳到指定游戏日 / 日内时段 |
| `/sre:60s tech` / `info` / `sacrifice` | 打开科技树（研究台亦可） / 查看队伍信息 / san 归零时自我解脱 |
| `/sre:client screen intro_sixty_seconds [all]` | 播放开场演出（`all`=OP 向全员播放） |
| `/sre:60s pve on\|off` | **OP**：PVE 开关（探索区游荡感染体 + 夜晚 Boss 尸潮领主；默认开，按图持久化） |
| `/sre:60s boss [1-5]` | **OP**：在自己面前立即刷指定等级 Boss 尸潮领主（测试/活动用） |
| `/sre:60s_area level <1-5>` | **OP**：设**全局探索区危险等级**——等级越高物资箱稀有物越常见、掷出件数越多，但游荡怪更多更强 |
| `/sre:60s_area level <1-5> <x y z>` | **OP**：设**包含该点的门绑定探索区**的危险等级（0=继承全局；可做「高危高收益」的深层区域） |
| `/sre:60s_area template <kind> <x1 y1 z1> <x2 y2 z2>` | 登记模板区域，填两个对角的绝对坐标（kind = residential/shelter/searchzone） |
| `/sre:60s_area spawn <kind> <x> <y> <z>` | 登记出生点（模板内绝对坐标） |
| `/sre:60s_area anchor <x y z>` / `anchor clear` | **OP**：登记/清除**避难所锚点门**（模板内那扇门的绝对坐标；配合下面的 `shelter_at_door`，见 §4.1）。手上有**锚点绑定工具**时用不着敲这条 |
| `/sre:60s npc add <变体> [档案] [半径]` | **OP**：在脚下登记一个 NPC 生成点并放预览立牌（=NPC 放置器右键） |
| `/sre:60s npc list` / `remove <序号>` / `clear` | **OP**：列出（点击传送）/ 删单个 / 清空 NPC 生成点 |
| `/sre:60s npc preview` / `hide` | **OP**：按配置重放全部预览立牌 / 收起预览（不动配置） |
| `/sre:60s npc profile <序号> <档案>` / `radius <序号> <n>` | **OP**：改已登记点的商人货架档案 / 驻守半径（=NPC 调校器右键） |
| `/sre:60s npc pirate` | **OP**：在脚下水面立刻放一船海盗（测试海上遭遇；脚下必须是水） |
| `/sre:60s autorevive on\|off` | **OP**：自动复活开关（**默认开**）——死亡后在本队避难所复活，尸体标在地图上、HUD 显示倒计时；开启时一波团灭不会直接判负 |
| `/sre:60s autorevive interval <秒>` | **OP**：复活间隔（**默认 240**=4 分钟；只影响此后的死亡，已在倒计时中的不受影响） |
| `/sre:60s shelter_at_door on\|off` | **OP**：避难所是否直接生成在各队探索区出口门上（**默认开**；缺锚点/门不够分则回退网格克隆） |
| `/sre:60s sea_teleport on\|off` | **OP**：海图是否允许扬帆传送与返回住所（**默认关**=玩家自己乘船；关闭时海图仍显示岛屿/庇护所/队友） |
| `/sre:60s_area grid <baseX> <baseY> <baseZ> <spacing>` | 每队网格偏移与间距 |
| `/sre:60s_area show` | 查看配置是否完整 |
| `/sre:area_manager set noReset true` | 地图设为不重置（模板不被覆盖） |

## 7. 物品速查

可 `/give @p noellesroles:<id>`（全部在创造模式的专属「**末日60秒**」标签页；方块另在方块页保留）：

| ID | 名称 | 作用 |
| --- | --- | --- |
| `sixty_seconds_water_small/_medium/_high` | 小/中/大瓶水 | 饮用恢复口渴（+15/+35/+60），不可堆叠 |
| `sixty_seconds_umbrella` | 雨伞 | 污雨时在野外（搜索区）持有可免除额外污染 |
| `sixty_seconds_crowbar` | 撬棍 | 右键强行闯入别队避难所（触发对方**报警**），一次性 |

## 8. 运行时流程与提示

- **准备阶段（60s）**：家中搜箱子 → 携带上限受限（父 8/其余 2，其余屏障）→ **右键门存进库存（潜行右键更快）** → 继续搜 → 时间到进避难所。HUD 有陨石倒计时。
- **游戏日**：饥饿/口渴/san 每分钟消耗、污染累积（**在家 ×0.5**）；饥渴清空后**每秒仅 -1 健康**（封顶不叠加）；污染满=缓慢+概率生病（不掉血）。
- **探索**：门菜单「外出探索」出门隐身 15s、归来冷却 120s；搜索区物资箱**惰性按天刷新**（仅交互到的箱子重掷，不做全图扫描）。
  右键物资箱触发 **3 秒「搜刮」进度条**（搜打撤式），完成才拿物资；离箱 >3 格 / 死亡会中断。
- **电脑**：避难所内小游戏每分钟换一个不同的，提示 20s 内完成（严格奖励门控见玩法文档限制）。
- **事件**：约每 3 分钟概率触发随机事件——**污雨**（无伞加速污染）/**浓烟**（伞无效，防毒面具可挡）/**寒潮**（饥饿加速）/**空投**；管理员也可 `/sre:60s airdrop` 手动投放。
- **拜访**：门菜单「拜访别队」选队 → 交易/进入避难所 → 目标队任一成员同意/拒绝（30s 超时）。
- **闯入**：撬棍（报警）/撬锁器（潜行）一次性进入别队避难所。

## 9. 常见问题

- **克隆出的实例重叠 / 互相穿模**：`grid spacing` 调大（要大于模板在 X 轴的尺寸 + 余量）；三种模板本身也要拉开。
- **开局没克隆 / 日志报「未配置完整」**：`/sre:60s_area show` 看 `complete`；六项（3 模板 + 3 出生点）缺一不可。
- **旧配置区域异常**：`sixty_seconds_config.json` 模板第二向量已改为**第二对角的绝对坐标**（字段名 `max`，旧名 `size` 兼容读取但按对角解释）；若此前按"各轴方块数"填写，用 `template` 命令重新登记一次即可（设置成功会回显归一化后的区域范围）。
- **模板被重置覆盖**：确认 `noReset=true`。
- **贴图/物品说明**：全部 60s 物品与方块已配专属贴图 + 名称 + tooltip 介绍描述；
  贴图由 `tools/gen_sixtyseconds_textures.py`（PIL）生成，改动后重跑该脚本即可。
- **还没做的**：随机商人（事件门业务）、多物品交易（当前主手对主手）。其余玩法均已落地，详见玩法文档 §10。
