# 地图场景快速使用指南

## 最简单的完整流程

1. 先载入或保存当前地图。
2. 执行 `/sre:scene wizard` 打开地图构建助手的“场景”页。
3. 在真实窗外建筑的两个对角分别点击“设置源区域最小角”和“设置源区域最大角”。
4. 点击“自动选择滚动轴”，再点击“刷新投影”。
5. 使用 XYZ 偏移，让紫色目标框和投影出现在绿色 `playArea` 周围的正确位置。
6. 点击“场景管理”，输入一个场景 ID，例如 `space_window`。
7. 点击“保存为新场景”。场景会保存到场景库，并自动指定给当前地图。
8. 返回地图助手，点击“一键发布并保存”。

以后其他地图需要复用同一个场景时，只需打开“场景管理”，选中 `space_window`，再点击“指定给当前地图”。

## 现在的数据关系

- 地图选择场景。
- 场景不再绑定某一张地图，可以被多张地图复用。
- 地图文件只保存：

```json
{
  "scene": "space_window"
}
```

- 场景配置保存于：

```text
<world>/scene_library/space_window.json
```

- 发布后的压缩资产保存于：

```text
<world>/scene_assets/<sha256>.sresc
```

旧地图中内嵌的 `sceneArea`、`sceneScroll` 和 `sceneAsset` 仍然可以读取。把旧场景保存到场景管理器后，再保存地图，即可迁移到新格式。

## 场景管理器按钮

- `指定给当前地图`：读取选中的场景，并把场景 ID 写入当前地图。
- `保存为新场景`：把当前选区、滚动轴、偏移和资产设置保存成新场景。
- `覆盖场景`：更新已经存在的场景。
- `删除场景`：删除未被当前地图使用的场景配置。
- `保存远程 URL`：设置 CDN 或 Web 资产地址。
- `可信快速模式`：跳过客户端完整哈希和注册表复扫。
- `取消地图场景指定`：让当前地图不再使用场景。
- `客户端场景：开/关`：只控制本机正式游戏中的移动场景。

## 客户端关闭移动场景

可以在客户端设置中关闭“启用地图移动场景”，也可以使用：

```text
/sreclient:scene enable
/sreclient:scene disable
```

关闭后不会注册或渲染正式游戏中的虚拟场景区块。客户端缓存和管理员投影仍然可用。

## 地图保存与导入

地图构建助手的“地图”页支持：

- 修改地图名称。
- 保存为新地图。
- 覆盖保存地图。
- 载入地图配置。
- 从 `<world>/map_imports` 导入 JSON。

导入时只填写文件名，不能填写任意磁盘路径。

## 常见故障

### 普通方块全部消失

先在场景管理器关闭“客户端场景”。关闭后场景渲染路径会立即卸载，不应影响普通地图区块。

### 场景显示“已过期”

正式游戏运行期间的方块变化不会再把场景资产误标为过期；重复写入相同方块状态也会被忽略。确实修改了源场景后，执行一次“一键发布并保存”。

### 场景只显示下半部分

重新发布资产。Y 轴循环会把虚拟区段安置到世界有效高度内，避免第二份循环落到最低高度以下被裁切。场景高度超过世界可容纳的双份高度时会自动改为静态显示并给出提示。
