## 现有系统分析

### 🎯 任务系统（Task System）
**当前状态：**

| 任务类型 | 完成方式 | 持续时间 |
|---------|---------|---------|
| 睡觉（SLEEP） | 躺在床上 | 8秒 |
| 阅读（READ_BOOK） | 打开讲台上的书 | 8秒 |
| 吃东西（EAT） | 吃零食 | 即时 |
| 喝饮料（DRINK） | 喝鸡尾酒 | 即时 |
| 锻炼（EXERCISE） | 在锻炼室的黑色混凝土上跑步 | 6秒 |
| 冥想（MEDITATE） | 蹲下保持静止 | 10秒 |
| 洗澡（BATHE） | 站在水中或花洒下 | 10秒 |
| 音符盒（NOTE_BLOCK） | 右键点击音符盒 | 10次点击 |

| 座椅休息（CHAIR） | 坐在椅子上 | 8秒 |

**任务生成机制：**
- 首个任务在游戏开始30秒后生成
- 任务间隔：30秒 ~ 60秒（随机）
- 权重系统：`1 / timesGotten` —— 已获得次数越多的任务权重越低
- 同一时间只能有一个任务
- 完成奖励：+0.5 心情值 + 50金币（无辜/中立角色）

### 😤 情绪系统（Mood System）
- 范围：0.0 ~ 1.0
- 每个未完成的任务使情绪以 `1/240 ticks/tick` 的速率下降
- 完成任务恢复 0.5 情绪
- 情绪阈值：
  - **中等** < 0.55：其他玩家手持物品产生幻觉
  - **抑郁** < 0.20：移速降低20%
  - **愤怒** > 0.75：移速提高15%

## 问题诊断：任务过于死板

### 🔴 核心问题

1. **任务间隔固定且机械化**
   - 任务冷却在30~60秒之间随机，但变化范围太小
   - 玩家很快就能预判下一个任务的到来时间
   - 缺乏紧张感和惊喜感

2. **任务奖励单一且固定**
   - 每次完成任务都是固定的 +50 金币 + 0.5 情绪
   - 没有连续完成任务的额外激励
   - 没有速度奖励或效率奖励
   - 杀手角色仅获得微薄的 +5 金币，缺乏做任务的动力

3. **任务类型行为重复**
   - 大多数任务只是"去某处站/坐一段时间"
   - 缺少与其他玩家互动的任务
   - 没有与当前游戏局势相关的动态任务

4. **任务与游戏进程脱节**
   - 无论游戏进行到哪个阶段，任务的生成和奖励都完全一样
   - 游戏后期与前期没有任何差异化体验

## 问题诊断：游戏公式化

### 🔴 核心问题

1. **收入曲线过于线性**
   - 被动收入恒定（每10秒+5），任务奖励恒定（+50）
   - 金币积累是纯粹的时间函数，没有策略性

2. **情绪系统影响力不足**
   - 情绪只影响移速和幻觉，对商店、任务无影响
   - 玩家缺少主动管理情绪的动力

3. **缺乏"风险-回报"权衡**
   - 做任务毫无风险，只有好处
   - 没有"选择"——拿到任务就去完成，没有策略深度

## 解决方案一：动态任务系统

### ✅ 已实现：基于游戏进程的动态任务冷却

**改动：** `GameConstants.java` + `SREPlayerTaskComponent.java`

```
核心改动：
- 任务冷却时间根据游戏已过时间动态缩短
- 游戏前期（前2分钟）：正常冷却 30-60秒
- 游戏中期（2-5分钟）：冷却缩短至 25-50秒
- 游戏后期（5分钟后）：冷却缩短至 20-40秒
```

**设计意图：**
- 游戏前期让玩家有探索和了解环境的时间
- 游戏后期任务更频繁，增加紧迫感和策略深度
- 杀手需要在更频繁的任务中找到下手机会
- 平民在后期有更多理由在列车中移动，增加遭遇概率

## 解决方案二：任务连击奖励

### ✅ 已实现：连续完成任务获得递增金币奖励

**改动：** `SREPlayerTaskComponent.java` + `RoleMethodDispatcher.java`

```
核心逻辑：
- 新增 taskStreak 字段追踪连续完成的任务数
- 连击奖励公式：baseReward + (streak * 5) 金币
  - 第1个任务：50 + 0 = 50金币
  - 第2个连续任务：50 + 5 = 55金币
  - 第3个连续任务：50 + 10 = 60金币
  - 最高额外奖励上限：+25金币（5次连击封顶）
- 超时未完成任务（情绪过低）会重置连击计数
```

**设计意图：**
- 鼓励玩家持续做任务，而不是偶尔做一个
- 给玩家一个"要不要冒险去做下一个任务"的决策点
- 杀手可以利用高连击的平民——他们更有可能为了保持连击而冒险走向任务点

## 解决方案三：情绪驱动的任务权重

### ✅ 已实现：低情绪时更容易生成简单/安抚性任务

**改动：** `SREPlayerTaskComponent.java`

```
核心逻辑：
- 当情绪 < 0.55（中等阈值）时：
  - 冥想、睡觉、座椅休息等"安抚性"任务的权重翻倍
  - 运动、外出等"活跃性"任务权重降低
- 当情绪 > 0.75（愤怒阈值）时：
  - 运动、外出等任务权重提升
  - 静态任务权重降低
```

**设计意图：**
- 任务不再完全随机，而是与玩家当前状态呼应
- 情绪低落时系统会"关心"玩家，提供更容易完成的任务
- 情绪亢奋时提供更有挑战性的任务作为发泄途径
- 增加情绪系统在游戏中的存在感和意义

## 综合改进建议

### 🎨 短期可实现（本次已部分实现）

| 改进点 | 状态 | 影响 |
|-------|------|------|
| 动态任务冷却 | ✅ 已实现 | 游戏后期节奏加快 |
| 任务连击奖励 | ✅ 已实现 | 增加策略选择 |
| 情绪驱动任务权重 | ✅ 已实现 | 任务不再千篇一律 |

### 🚀 中期建议（未来可以考虑）

1. **限时任务系统**
   - 任务有完成时限（如60秒内完成）
   - 限时内完成额外奖励
   - 超时自动取消并扣除情绪

2. **双人协作任务**
   - 某些任务需要两个玩家在附近才能完成
   - 增加玩家之间的互动和信任/怀疑

3. **商店限时折扣**
   - 随机出现短暂的商品打折
   - 给玩家创造"要不要现在买"的决策

4. **危险区域与安全区域**
   - 地图上标记临时的"危险区域"和"安全区域"
   - 在安全区域做任务获得额外奖励
   - 在危险区域停留会降低情绪

5. **任务拒绝机制**
   - 允许玩家跳过不想做的任务（消耗少量金币）
   - 给玩家更多自主权

### 🌟 长期展望

1. **事件系统（Random Events）**
   - 列车晃动、灯光闪烁、广播通知等随机事件
   - 影响全局的短期效果

2. **声望系统**
   - 完成任务积累声望
   - 高声望玩家被投票时获得额外辩护机会

3. **物品改良系统**
   - 连续完成任务后武器/工具获得临时增强
   - 不同任务完成组合触发不同效果

## 技术实现概要

### 文件改动清单

| 文件 | 改动说明 |
|------|---------|
| `GameConstants.java` | 新增 `getTaskCooldown()` 方法，支持基于游戏时间的动态冷却 |
| `SREPlayerTaskComponent.java` | 添加 `taskStreak` 字段和连击逻辑；改动 `generateTask()` 添加情绪权重 |
| `RoleMethodDispatcher.java` | 修改 `callOnFinishQuest()` 使用连击奖励公式 |
| `zh_cn.json` | 新增连击相关的翻译文本 |

### 向后兼容性

- 所有改动都是增量式的，不会破坏现有存档
- `taskStreak` 默认为0，不影响现有NBT数据
- 动态冷却保持在原有范围附近，不会让游戏失衡
- 配置项保持不变，所有新功能使用合理默认值
